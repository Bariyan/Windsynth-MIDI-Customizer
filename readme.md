# Windsynth MIDI Customizer (LV2)

ウィンドシンセサイザーの演奏表現を最適化し、音源ごとの特性に合わせるための LV2 MIDI ユーティリティプラグインです。
ピッチベンドの再マッピング、ブレス連動型の擬似ダイナミックベロシティ、およびトランスポーズ機能を提供します。

* **Plugin URI**: `urn:bariyan:wsyn-midi-customizer`
* **バンドル名**: `wsyn_midi_customizer.lv2`
* **動作確認ホスト**: **Carla**

## 主な機能

### 1. ピッチベンドの再マッピング
入力されたピッチベンドを、以下のパラメータに基づいてリアルタイムで変換します：
* **Calibration Center**: 楽器個体ごとの「中心のズレ」を補正する基準点を設定します。
* **DeadZone**: 中心付近の不感帯を設定し、不要なピッチの揺れを抑えます。
* **Reach Distance**: 上下独立した到達距離設定により、少ない操作量（噛み込み量や親指の動き）で最大ベンドまで到達させることができます。
* **応答曲線の選択**: 上下それぞれに線形（Linear）、指数（Exponential）、対数（Logarithmic）のカーブを適用し、好みのかかり具合に調整できます。

### 2. Pseudo Dynamic Velocity（擬似ダイナミックベロシティ）
ほとんどのウィンドコントローラーでは本体にノートオン時のダイナミックベロシティ(発音時の息量をベロシティにする)が設定できますが、本機能の初制作者はこれを利用していません。多くの音源ではこの機能が音色のエンベローブのアタックを決定してしまい、あとからブレスを上げても思うようにサステイン音量が上がらず、管楽器らしい表現にならないからです。

しかし、打楽器や撥弦楽器(ギターなど)の音色をウィンドコントローラーで演奏しようという無謀なことを考えた場合、やはりダイナミックベロシティが必要になるでしょう。こういった場合、本来は本体の設定を都度変えるべきなのでしょうが、音色を変えるごとに本体設定も変えるというのはなかなか大変です。

本機能はコントローラーから送られてくるノートオンとブレス（CC#2）を用いて、擬似的にダイナミックベロシティを実現しようとしています。
* **動作**: 設定値が 1 以上の場合、ノートオン直前のブレス（CC#2）の値をキャッチし、それをノートオンのベロシティとして上書きします。
* **Floor機能**: 上書き後のベロシティが設定値を下回らないように底上げし、吹き始めの音が消えたり弱すぎたりするのを防ぎます。
* **OFF設定**: 0 に設定するとこの機能は無効化され、ノートオンに付された元のベロシティを維持します。

### 3. トランスポーズ
* アルトサックスに合わせてコントローラーはEb運指にしているが、ソプラノサックス音色のときはBb運指にして雰囲気を味わいたい、でも本体設定は変えたくないというわがままな要請に応えるべく、ノートオン／オフイベントを指定した半音単位でシフトします（±3オクターブ）。出力は 0〜127 の範囲にクランプされます。

## パラメータ仕様

| 名前 | シンボル | 説明 |
| :--- | :--- | :--- |
| **Transpose** | `transpose` | 半音単位の移調 (-36..36) |
| **Calibration Center** | `center_in` | 楽器側のピッチベンド中心値 (0..16383) |
| **DeadZone** | `deadzone` | 中心付近の無反応領域 (±) |
| **Upper Reach** | `dist_u` | 上方向に最大値へ到達するまでの距離 |
| **Lower Reach** | `dist_l` | 下方向に最大値へ到達するまでの距離 |
| **Upper Curve** | `curve_u` | 上側のカーブ種別と深さ (0=Linear, 1-5=Exp, 6-10=Log) |
| **Lower Curve** | `curve_l` | 下側のカーブ種別と深さ (0=Linear, 1-5=Exp, 6-10=Log) |
| **Pseudo Dynamic Velocity**| `v_floor` | 0=無効, 1-127=有効（CC#2連動＋最小値設定） |
| **Channel**| `channel` | 処理対象チャネル 0=全て, 1-16=対象のチャネル |
| **Output Capacity**| `out_capacity` | 処理バッファサイズ 0=auto, 1-16=4096～65536 |

## ビルドとインストール

### 必要条件
* GCC (C11 対応)
* LV2 開発用ヘッダ

### 手順
```sh
# ビルド
make

# インストール (~/.lv2/ 配下へコピー)
make install
```
